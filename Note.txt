Online Shop - Tam Implementation Roadmap
FAZA 1 — Kritik Eksikler (Önce Bunlar)
1. Sipariş Onay E-postası
Ne yapılacak: Stripe webhook'u ödeme tamamlandığında kullanıcıya otomatik e-posta göndermeli.

Adımlar:

A) lib/mail.ts dosyasına yeni fonksiyon ekle:


sendOrderConfirmationEmail(to, {
  orderNumber,
  items: [{ title, quantity, price }],
  total,
  shippingAddress
})
Mevcut sendResetPasswordEmail fonksiyonuyla aynı Resend pattern'i kullanacak. HTML template içinde sipariş tablosu olacak.

B) app/api/stripe/webhook/route.ts dosyasını güncelle:
Mevcut payment_intent.succeeded event handler'ına şu satırları ekle:


- Order'dan user bilgilerini çek (prisma ile)
- sendOrderConfirmationEmail() çağır
Bağımlılık: Mevcut Resend kurulumu var, sadece yeni template + çağrı ekleniyor.

2. Kategori Yönetimi (Admin UI)
Ne yapılacak: Admin paneline kategori ekleme/silme/düzenleme ekranı.

Adımlar:

A) Yeni API rotaları oluştur:

app/api/admin/categories/route.ts → GET (liste), POST (ekle)
app/api/admin/categories/[id]/route.ts → PATCH (güncelle), DELETE (sil)
Pattern olarak mevcut app/api/admin/banners/ route'larını baz al.

B) Yeni admin sayfası:

app/admin/categories/page.tsx → Server component, kategorileri listele
components/admin/CategoryManagementCard.tsx → CRUD operasyonları için Client component
C) Mevcut app/admin/page.tsx'e link ekle — diğer admin kartları gibi.

DB değişikliği yok — Category modeli zaten Prisma schema'da mevcut.

3. İndirim Kodu / Kupon Sistemi
Ne yapılacak: Admin kupon oluşturur, kullanıcı checkout'ta girer, sepet toplamından düşer.

Adımlar:

A) Prisma schema'ya yeni model ekle:


model Coupon {
  id          String   @id @default(cuid())
  code        String   @unique
  type        CouponType  // PERCENTAGE veya FIXED_AMOUNT
  value       Int      // % veya cent cinsinden
  minAmount   Int?     // minimum sepet tutarı (cent)
  maxUses     Int?     // kaç kez kullanılabilir
  usedCount   Int      @default(0)
  expiresAt   DateTime?
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now())
  usages      CouponUsage[]
}

model CouponUsage {
  id        String   @id @default(cuid())
  coupon    Coupon   @relation(...)
  user      User     @relation(...)
  order     Order    @relation(...)
  createdAt DateTime @default(now())
  @@unique([couponId, userId])  // kişi başı 1 kullanım
}
Ayrıca Order modeline couponId, discountAmount alanları ekle.

B) Yeni API rotaları:

app/api/coupon/validate/route.ts → POST, kodu doğrula + indirim hesapla
app/api/admin/coupons/route.ts → GET/POST
app/api/admin/coupons/[id]/route.ts → PATCH/DELETE
C) Checkout sayfasına kupon input alanı ekle:

components/CouponInput.tsx → Kod gir → Validate API çağır → İndirim göster
app/(main)/cart/checkout/page.tsx'e CouponInput ekle
D) Order oluşturma mantığını güncelle:

app/api/orders/create/route.ts'e kupon validasyonu ekle, transaction içinde usedCount++ yap
E) Admin kupon yönetim sayfası:

app/admin/coupons/page.tsx
4. Misafir Checkout (Hesapsız Alışveriş)
Ne yapılacak: Kullanıcı hesap açmadan, sadece e-posta + adres girerek ödeme yapabilmeli.

Adımlar:

A) Prisma schema güncelle:
Order modeline opsiyonel misafir alanları ekle:


guestEmail     String?
guestFirstName String?
guestLastName  String?
guestPhone     String?
// userId zaten optional yap
B) Sepet için session-based geçici cart:

Misafirler için cookie/localStorage tabanlı geçici sepet
context/GuestCartContext.tsx yeni context
hooks/useCart.ts → hem auth hem guest cart'ı birleştirsin
C) Checkout flow'u ikiye ayır:

Auth kullanıcı → mevcut flow devam
Misafir → form: e-posta, ad, soyad, telefon, adres
app/(main)/cart/checkout/page.tsx güncelle
D) Misafir order API:

app/api/orders/create/route.ts'i genişlet, session yoksa guest bilgilerini kullan
Not: Bu özellik en karmaşık olanı, diğerlerinden sonraya bırakılabilir.

13. İade Yönetimi
Ne yapılacak: Kullanıcı iade talep eder, admin onaylar/reddeder.

model ReturnRequest {
  id        String        @id @default(cuid())
  order     Order         @relation(...)
  user      User          @relation(...)

  status    ReturnStatus  // PENDING, APPROVED, REJECTED, REFUNDED
  adminNote String?
  createdAt DateTime      @default(now())
  updatedAt DateTime      @updatedAt
}

enum ReturnStatus { PENDING APPROVED REJECTED REFUNDED }
B) Kullanıcı tarafı:

FAZA 4 — Pazarlama & SEO
14. SEO Optimizasyonu
Ne yapılacak: Her sayfada doğru meta tag, Open Graph, sitemap.

Adımlar:

A) Her sayfa için generateMetadata() ekle:


// app/(main)/product/[id]/page.tsx
export async function generateMetadata({ params }) {
  const product = await getProduct(params.id);
  return {
    title: `${product.title} | My Store`,
    description: product.description,
    openGraph: {
      images: [product.thumbnail]
    }
  }
}
B) app/sitemap.ts oluştur:

Tüm ürün, kategori URL'lerini otomatik listele
C) app/robots.ts oluştur:

Arama motoru yönlendirme kuralları
D) Ürün sayfasına JSON-LD structured data ekle:


// Schema.org Product markup
